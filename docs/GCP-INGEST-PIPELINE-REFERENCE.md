# GCP Ingest Pipeline Reference

This document describes the ingest pipeline architecture for GCP services in Cloud Loadgen for Elastic.

## Overview

Each GCP generator produces documents in **vendor-native format** (matching Google Cloud Logging / Cloud Monitoring schemas). Ingest pipelines handle all **ECS normalization** — mapping GCP-native fields to Elastic Common Schema fields — so that dashboards, ML jobs, and alerting rules work correctly.

## Pipeline registry

All GCP pipelines are generated from the service catalog and stored in:

```
installer/gcp-custom-pipelines/pipelines/registry.mjs
```

The registry is generated by `npm run gen:gcp-pipelines` (via `scripts/generate-gcp-pipeline-registry.mjs`) and exports a `PIPELINE_REGISTRY` array.

## Pipeline naming convention

| Pattern | Example | Purpose |
| ------- | ------- | ------- |
| `logs-gcp.{dataset}-default` | `logs-gcp.cloudfunctions-default` | Log parsing and ECS normalization |
| `metrics-gcp.{dataset}-default` | `metrics-gcp.cloudfunctions-default` | Metric field mapping |

## Standard processors

Every GCP log pipeline includes these processor stages:

### 1. ECS normalization (`ecsNorm`)

Maps GCP-native fields to ECS equivalents:

| GCP native field | ECS field |
| ---------------- | --------- |
| `severity` | `log.level` |
| `protoPayload.methodName` | `event.action` |
| `protoPayload.status.code` | `event.outcome` (0 = success, else failure) |
| `resource.type` | `cloud.service.name` |
| `resource.labels.project_id` | `cloud.project.id` |

### 2. Identity extraction (`gcpIdentityExtract`)

| GCP native field | ECS field |
| ---------------- | --------- |
| `protoPayload.authenticationInfo.principalEmail` | `user.email`, `user.name` |
| `protoPayload.requestMetadata.callerIp` | `source.ip` |
| `protoPayload.requestMetadata.callerSuppliedUserAgent` | `user_agent.original` |

### 3. Related fields (`populateRelatedFields`)

Populates `related.ip` and `related.user` arrays from extracted identity fields.

### 4. Vendor cleanup (`vendorCleanup`)

Optionally removes GCP-native fields that have been mapped to ECS equivalents, keeping the document clean without losing information.

## Installing pipelines

```bash
# Via wizard
# Setup → Install Cloud Loadgen Integrations (GCP)

# Via CLI
npm run setup:gcp-pipelines

# Via loadgen packs (per-service)
npm run setup:gcp-loadgen-packs
```

## Regenerating the registry

If you add new GCP generators or modify field mappings:

```bash
npm run gen:gcp-pipelines
```

This reads the GCP elastic maps and generator metadata to produce the pipeline registry.
