# AWS Services — Documentation Review & Coverage

> **Last updated:** 2026-03-17 (v8.0)

This document reviews the load generator’s AWS services against **official AWS monitoring, logging, and metrics documentation**. It provides doc links, coverage summary, and gaps. For **Glue**, see [GLUE-METRICS-COVERAGE.md](GLUE-METRICS-COVERAGE.md). Historical gap-checklist material lived in legacy docs that are no longer shipped in this repo; use the generators and [GAP-ANALYSIS-LOGS-AND-METRICS.md](GAP-ANALYSIS-LOGS-AND-METRICS.md) for current context.

---

## How to use this review

- **Doc links** — Point to the main monitoring/metrics/logging pages for each service.
- **Coverage** — What the generator already emits (metrics, lifecycle, errors) vs what AWS documents.
- **Gaps** — Missing metrics or log patterns that would align with AWS docs.
- **N/A** — Services with no dedicated CloudWatch metrics (e.g. some security/audit services use CloudTrail or custom logs).

## Summary: high-priority gaps

1. **EMR** — Add run_state, job-run message signals, and Glue-style metrics (elapsedTime, tasks, JVM/GC, numberAllExecutors). See [enhancement-candidates.md](enhancement-candidates.md).
2. **Batch** — Add “Job run started/succeeded/failed” in messages and elapsedTime (or Duration) in metrics.
3. **DataBrew** — Add run_state, message signals, and `aws.databrew.metrics` block.
4. **AppFlow** — Add message signals and `aws.appflow.metrics` block.
5. **CodeBuild** — Add “Build started/succeeded/failed” in message pool.
6. **Athena** — Add “Query started/succeeded/failed” in message pool.
7. **Lambda** — Optional: DestinationDeliveryFailures, granular throttle metrics, async metrics (AsyncEventsReceived, AsyncEventAge, AsyncEventsDropped), provisioned concurrency metrics. Confirm event.duration unit (ms vs µs).

For **Glue**, coverage is documented in [GLUE-METRICS-COVERAGE.md](GLUE-METRICS-COVERAGE.md). For **all services**, baseline ECS alignment (event.duration, error on failure, dimensions) is in [PLAN-ALL-SERVICES.md](PLAN-ALL-SERVICES.md).
